🎯 设计目标

数据维度 —— 从内存到全量

  • 内存 history[100] 保留做实时展示
  • 新增 GET /ocgt/api/stats 读取 JSONL 日志
  • 支持今日 / 7天 / 30天 / 自定义时间范围

可视化 —— 从数字到图表

  • Token 消耗趋势折线图 (每日)
  • 请求量柱状图 + 成功率叠加
  • 模型分布环形图
  • 客户端来源玫瑰图

交互 —— 从静态到动态

  • 时间范围选择器 (今日/7天/30天/自定义)
  • 多维度下钻 (点击模型 → 只看该模型)
  • 搜索 + 高级筛选

导出 —— 从只看不能拿

  • CSV 导出 (表格数据)
  • JSON 导出 (原始数据)
  • 支持当前筛选结果导出

🔧 关键技术决策

图表库

Chart.js v4 — 轻量 (70KB)、零依赖、中文文档完善、支持折线/柱状/环形图。从 CDN 加载，无需 npm 构建。

JSONL 读取策略

每日文件 `ocgt-*.jsonl` 逐行读取，按时间范围过滤。一天 ~500KB × 30天 = ~15MB，全读入内存后再聚合，Go 处理 < 200ms。

前端状态管理

当前使用全局变量 + DOM 直接操作。新增 Chart.js 实例存储在全局，切换时间范围时 `chart.destroy()` + 重建。

Cost 估算

根据模型价格表（hardcode 常见模型 $/1K tokens）估算花费。标注 "估算" 避免误导。

📐 视觉风格参考

流量监控页将采用与现有 ocgt 仪表盘一致的设计语言：

  • 配色 — 沿用现有 accent color (#3b6eff)，Token 相关用绿色系 (#3fb950)，错误用红色系 (#d63638)
  • 暗色优先 — 默认深色背景，与 Chart.js 深色主题一致
  • 图表风格 — Chart.js 使用圆角线条、渐变填充、半透明网格
  • 响应式 — 图表在大屏并排展示，小屏堆叠
  • 交互细节 — 图表 hover 显示 tooltip，可点击图例切换显示/隐藏系列
